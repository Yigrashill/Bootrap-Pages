1. Programowanie Obiektowe

Programowanie obiektowe to koncepcja programowania bazująca na pojęciach klasy i obiektu. Program składa się z wielu obiiektów komunikujących się między sobą w celu dokonania zadania.

3. Modifikatory Dostępu

  • Public - publiczne składniki klasy są dostępne dla wszystkich metod wszystkich klas.
  • Private - składowe prywante są dostępne tylko dla metod klasy, w której się znajdują.
  • Protected - składowe dostępne są dla klasy w której się znajdują, oraz dla klas dziedziczącej po niej w danej bibliotece.
  • Internal - składowe sostępne wewnętrznie dla danej biblioteki (.dll-ki).

4. Pozostałe modyfikatoy klass

  • Abstract - klasa abstrakcyjna może być tylko klasą bazowaą dla innych klas. Nie można utworzyć instancji takiej klasy.
  • Async - wskazuje, że metoda, wyrażaenie lambada lub metoda anonimowa jest asynchricznie.
  • Const - oznacza, że wartość składowe lub zmiennej nie możę być modyfikowana.
  • Event - delkaruje zdarzenie.
  • Extern - wskazuje, żę metoda została zaimpleętowana zewnętrzenie.
  • New - ukrywa składniki klasy, które mają taką samą nazwę w klasie podstawowej.(operatory niweluje ostrzeżęnie o ukrywaniu skłdnika klasy).
  • Override - służy do nadpisywania metod wirtualnych odziedziczonej po klasie bazowej.
  • Partial - pozwala rozbić klasy, metody i struktury na wiele kawałków kodu przechowywanych w kilku plikach.
  • Readonly - określa, żę polu można nadać wartość jedynie w konstruktorze lub deklaracji pola.
  • Saled - określa, że klasa jest klasą ostateczną i nie może być klasą bazową dla innych klas.
  • Static - określa, że składnik nie należy do obiektu, tylko do typu.
  • Unsafe - deklaracje niebezpieczny kontekst, czyli dostęp do opertorów sizeod, q -?. oraz *.
  • Virtual - określa metody wirtualne, które mogą być nadpisane w klasach dziedziących po klasie bazowej, w której się znajdują.
  • Volatile - określa pole, które może być modyfikowane w programanie przez system operacyjny, sprzęt lub równolegle działający wątek.

5. Obiekt

Obiekt jest instancją klasy, czyli toworem o określonym zachowaniu i cechach zdefiniowanych w klasie.

W programowaniuy obiektowym nie działamy na klasie lecz na obiekcie, czyli instancji klasy.

Każdy obiekt charakteryzuje:
  • niepowtarzalność - każdy obiekt jest osbobnym tworem, możne jednak skopiować obiek, a dokładniej jego zawartość,
  • zachowanie - każdy obiekt może wykonać określone operacje,
  • stan w jakim się obiekt znajduje.

Pola

Pola to zmienne dowolnego typu dostępu w obrębie klasy. W zależności od modyfikatora dostępu dostępne są również dla innych klas. domyślnie pola są prywatne

                            
                                    [Modyfikator] Typ Identyfikator [= war początkowa]
                                    private int Wiek =5;

    modyfikator - dozwolene są modyfikatory: static, interrnal protected

    identyfikator - nzawa pola.

    wartość początkowwa - wart. początkowa pola.

Metody

Metody to funkcje, czyli bloki kodu zawierające serie instrukcji. Metoda jest skłądnikiem klasy implementującym obliczania albo działnia wywołane przez obiekt.

Moetoda musi zawracać wartość zgodnią z tytułem umieszczanym w defincji. Aby zwrócić wartośż i zakończyć działanie metody, używa się słowa kluczowego return. Dalsza część kodu znajdująca się za słowem return nie zostanie wykoanana.

W przypadku metod niezwacajćych żadnych wartości słowo kluczowe nie jest wymagane.

Tworząc metodą można zawierać wiele instrukcji retrun dla kaego scenariusza użycia, przy czym będzie wywołana tylko pierwsza.

                                Public class Punkt
                                {
                                    private int _x;

                                    public int Get(x)
                                    {
                                        return x;
                                    }

                                    public void SetX(int x)
                                    {
                                        _x = x;
                                    }
                                }

Całość powyższego kodu można zastąpić właściwością

                                public Class Punkt
                                {
                                    public int x { get; set;}
                                }

Całość działa tak samo dzięki automatycznym sete -rem i getreturn.
